# 보류 개발 항목

세션 간 유지되는 백로그. planner가 제안했으나 우선순위에서 밀린 항목.
새 세션 시작 시 이 파일을 읽고 현재 진행할 항목을 판단한다.

## 작성 규칙
- planner 제안 중 보류된 항목을 즉시 기록
- 항목별로 출처, 보류 사유, 우선순위 표기
- 착수 시 "진행중"으로 변경
- 완료 시 이 파일에서 삭제하고 dev_his/summary-YYYY-MM.md 로 이동

## 백로그

### 높음

(예시)
- [ ] 소셜 로그인 (Google, Kakao)
  - 출처: 2026-04-08 인증 설계 세션
  - 사유: 이메일 로그인 먼저 구현
  - 참조: docs/engine/engine-spec-auth.md

### 중간

(예시)
- [ ] 상품 이미지 자동 리사이징
  - 출처: 2026-04-08 상품 설계 세션
  - 사유: MVP에서 제외
  - 참조: docs/be/be-spec.md

### 낮음

(예시)
- [ ] 판매자 대시보드 매출 차트
  - 출처: 2026-04-08 UI 설계 세션
  - 사유: 주문 목록으로 우선 대체
  - 참조: docs/ui/ui-spec.md
