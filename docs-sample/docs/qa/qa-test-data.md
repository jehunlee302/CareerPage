# QA 테스트 데이터 규칙

## 픽스처
- 테스트 데이터는 tests/fixtures/ 에 JSON/TS로 관리
- 실제 사용자 데이터 사용 금지
- 경계값 포함: 빈 문자열, null, 매우 긴 문자열, 특수문자, 유니코드

## 시드 데이터
- 개발 DB용: prisma/seed.ts
- 판매자 3명, 구매자 5명, 상품 20개, 주문 10개 (다양한 상태)
- 시드는 멱등성 보장 (재실행 시 충돌 없음)

## 테스트 격리
- 각 테스트는 독립적 (순서 의존성 금지)
- 통합 테스트: 각 테스트 전 DB 트랜잭션 시작, 후 롤백
- E2E: 각 시나리오 전 시드 초기화
