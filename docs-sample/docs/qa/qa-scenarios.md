# QA 테스트 시나리오

## 인증 시나리오
- 정상 회원가입 → 로그인 → 토큰 발급 확인
- 중복 이메일 가입 → 에러 응답 확인
- 잘못된 비밀번호 → 401 + 실패 카운트
- 토큰 만료 → 리프레시 → 새 토큰 발급
- 리프레시 토큰 재사용(탈취) → 모든 세션 무효화

## 결제 시나리오
- 정상 결제 → 주문 상태 PAID 확인
- 결제 실패 → 주문 상태 PENDING 유지
- 부분 환불 → 환불 금액 검증
- 중복 환불 시도 → 에러
- Stripe 웹훅 재전송 → 멱등성 확인

## 주문 시나리오
- 재고 0 상품 주문 → 에러
- 동시 주문 (동일 상품 마지막 1개) → 하나만 성공
- SHIPPED 후 취소 시도 → 에러
- 주문 상태 타임라인 정확성

## UI 시나리오 (E2E)
- 로그인 → 상품 검색 → 주문 → 결제 완료 (Happy Path)
- 네트워크 끊김 → 에러 화면 → 재시도 → 복구
- 모바일 뷰포트에서 전체 플로우
