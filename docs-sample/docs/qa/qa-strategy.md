# QA 전략

## 테스트 피라미드
- 단위 테스트: 70% (도메인 로직, 유틸리티)
- 통합 테스트: 20% (API 엔드포인트, DB 연동)
- E2E 테스트: 10% (핵심 유저 플로우)

## 환경
- 단위: 인메모리 (모킹)
- 통합: Docker Compose (PostgreSQL + Redis)
- E2E: Playwright (Chromium headless)

## 실행 타이밍
- 단위/통합: 매 커밋 (CI)
- E2E: 매 PR, 배포 전
- 성능: 주 1회 (별도 파이프라인)

## 품질 게이트 (CI 통과 조건)
- 전체 테스트 통과
- 커버리지 최소 80% (신규 코드 90%)
- 린트 에러 0건
